## Hallazgos detallados

### A01 вҖ” Broken Access Control В· рҹ”ҙ CRITICAL

**H-01.1 рҹ”ҙ Token de sesiГіn admin es predecible y sin validaciГіn server-side**

Archivo: [lib/admin-auth.ts:16](lib/admin-auth.ts) lГӯnea 16:
```ts
const token = Buffer.from(`${Date.now()}-${Math.random().toString(36)}`).toString('base64')
```

Problemas:
- `Date.now()` + `Math.random()` generan tokens adivinables (no crypto-safe)
- Base64 no es cifrado вҖ” reversible en 1 segundo
- Peor aГәn: en lГӯnea 28-32, `isAdminAuthenticated()` solo valida `!!session?.value`:
  ```ts
  export async function isAdminAuthenticated(): Promise<boolean> {
    const cookieStore = await cookies()
    const session = cookieStore.get(ADMIN_COOKIE)
    return !!session?.value  // вҡ пёҸ cualquier valor no vacГӯo pasa
  }
  ```

**Exploit:** cualquier atacante puede saltarse el login admin enviando el cookie:
```
Cookie: preenvios_admin_session=anything
```
Acceso total a `/api/admin/*` garantizado.

**Fix requerido:**
1. Generar token con `crypto.randomBytes(32).toString('hex')` en `createAdminSession()`
2. Almacenar el token vГЎlido server-side (tabla `admin_sessions` en Supabase con `token`, `expires_at`) o firmar con JWT + `JWT_SECRET`
3. En `isAdminAuthenticated()`, validar que el token estГЎ en la tabla Y no expirado

---

**H-01.2 рҹ”ҙ `NEXT_PUBLIC_CRON_SECRET` expuesto en bundle cliente**

Archivo: [app/[locale]/admin/panel.tsx:61](app/%5Blocale%5D/admin/panel.tsx#L61):
```tsx
const res = await fetch('/api/admin/dashboard', {
  headers: { authorization: `Bearer ${process.env.NEXT_PUBLIC_CRON_SECRET || ''}` }
})
```

Problema:
- Variables con prefijo `NEXT_PUBLIC_*` se inlinean en el JavaScript del bundle que se envГӯa al navegador
- Cualquier visitante de `/es/admin` puede abrir DevTools вҶ’ Sources y leer el valor
- Con ese secreto un atacante puede invocar `/api/scrape` y disparar scrapers, forzar 3-strikes failure y envenenar datos de todos los operadores

**Fix requerido:**
1. `app/api/admin/dashboard/route.ts` debe validar sesiГіn admin (`isAdminAuthenticated()`), no `CRON_SECRET`
2. Eliminar `NEXT_PUBLIC_CRON_SECRET` de `.env.local` y del Vercel env
3. El `panel.tsx` hace fetch sin necesitar Authorization header вҖ” la cookie de sesiГіn basta
4. `CRON_SECRET` (sin `NEXT_PUBLIC_`) queda exclusivamente para `/api/scrape` invocado por Vercel Cron

### A02 вҖ” Cryptographic Failures В· рҹ”ҙ CRITICAL

**H-02.1 рҹ”ҙ `ADMIN_PASSWORD` comparado en texto plano**

Archivo: [lib/admin-auth.ts:11-13](lib/admin-auth.ts):
```ts
export async function validateAdminLogin(email: string, password: string): Promise<boolean> {
  return email === process.env.ADMIN_EMAIL && password === process.env.ADMIN_PASSWORD
}
```

Problema:
- Si el `.env.local` filtra (backup comprometido, log leak, git error, misconfiguraciГіn Vercel), el password queda expuesto
- No hay bcrypt/argon2 hashing
- Supabase Auth en Fase 4.4.B mitiga esto, pero hasta entonces es vulnerable

**Fix requerido:**
1. `npm install bcryptjs` + `npm install -D @types/bcryptjs`
2. Generar hash: `node -e "console.log(require('bcryptjs').hashSync('mi-password-real', 12))"`
3. Guardar `ADMIN_PASSWORD_HASH` en Vercel env vars (remover `ADMIN_PASSWORD` plaintext)
4. En `validateAdminLogin`: `return email === ENV.ADMIN_EMAIL && bcrypt.compareSync(password, ENV.ADMIN_PASSWORD_HASH)`

6 usos encontrados (blog, wiki, paГӯs, operador, tasa, page SEO). Todos renderizan Schema.org JSON-LD derivado de datos de la DB o arrays estГЎticos вҖ” nunca input de usuario directo. вң“

**H-03.4 рҹҹЎ Webhook Twilio NO valida firma HMAC**

Archivo: [app/api/whatsapp/webhook/route.ts:60-81](app/api/whatsapp/webhook/route.ts):
```ts
const body = await request.formData()
const from = body.get('From')
const text = body.get('Body')
// ... procesa sin verificar X-Twilio-Signature
```

Problema: cualquier atacante puede POST al webhook fingiendo ser Twilio, disparar lГіgica del bot, abusar del endpoint para enumerar corredores, o inyectar respuestas falsas en la cola.

MitigaciГіn parcial: el bot no ejecuta acciones con efectos secundarios graves (solo consulta tasa + responde). Pero el riesgo sube si en el futuro se agrega logging de mensajes o integraciГіn con Supabase.

**Fix requerido** (TROUBLESHOOTING/16 ya documenta el patrГіn):
```ts
import { validateRequest } from 'twilio'
const signature = request.headers.get('x-twilio-signature') || ''
const url = process.env.TWILIO_WEBHOOK_URL!
const params = Object.fromEntries(body.entries())
if (!validateRequest(process.env.TWILIO_AUTH_TOKEN!, signature, url, params)) {
  return NextResponse.json({ error: 'invalid signature' }, { status: 401 })
}
```

### A04 вҖ” Insecure Design В· рҹҹЎ MODERATE

**H-04.1 рҹҹЎ Sin rate limiting en endpoints pГәblicos**

Endpoints afectados:
- `POST /api/suscripcion-free` вҶ’ permite spam masivo de altas + floods vГӯa Resend
- `POST /api/contactos` вҶ’ permite flood de mensajes en la tabla `contactos`
- `POST /api/admin/auth` вҶ’ brute force ilimitado (tambiГ©n A07)

**Fix requerido:**
- Instalar `@upstash/ratelimit` + `@upstash/redis` (plan free cubre ~10K req/dГӯa)
- LГӯmites sugeridos:
  - `/api/suscripcion-free`: 3 req / 10 min por IP
  - `/api/contactos`: 5 req / 10 min por IP
  - `/api/admin/auth`: 5 intentos / 15 min por IP + respuesta `429 Too Many Requests`

**H-04.2 рҹҹў Double opt-in implementado**

`/api/suscripcion-free` crea suscripciГіn con `confirmado: false` y solo activa tras GET con token vГЎlido вң“

### A05 вҖ” Security Misconfiguration В· рҹҹЎ MODERATE

**H-05.1 рҹҹЎ Faltan headers de seguridad globales**

[next.config.ts](next.config.ts) no configura headers. El sitio hoy sirve sin:
- `Strict-Transport-Security` (HSTS)
- `X-Content-Type-Options: nosniff`
- `X-Frame-Options: DENY`
- `Content-Security-Policy` (CSP)
- `Referrer-Policy: strict-origin-when-cross-origin`

**Fix requerido** (TROUBLESHOOTING/18 ya lo describe). Agregar a `next.config.ts`:
```ts
async headers() {
  return [{
    source: '/:path*',
    headers: [
      { key: 'Strict-Transport-Security', value: 'max-age=63072000; includeSubDomains; preload' },
      { key: 'X-Content-Type-Options', value: 'nosniff' },
      { key: 'X-Frame-Options', value: 'DENY' },
      { key: 'Referrer-Policy', value: 'strict-origin-when-cross-origin' },
      { key: 'Content-Security-Policy', value: [
        "default-src 'self'",
        "script-src 'self' 'unsafe-inline' https://www.googletagmanager.com",
        "style-src 'self' 'unsafe-inline' https://fonts.googleapis.com",
        "font-src 'self' https://fonts.gstatic.com",
        "img-src 'self' data: https://flagcdn.com https://cdn.brandfetch.io https://www.google-analytics.com",
        "connect-src 'self' https://*.supabase.co https://www.google-analytics.com",
        "frame-ancestors 'none'",
      ].join('; ') },
    ],
  }]
}
```

**H-05.2 рҹҹЎ Mensajes de error podrГӯan filtrar info de schema**

Varios `catch` devuelven `error.message` directo de Supabase, que puede incluir nombres de columnas o constraints. No es alto riesgo pero recomendable sanitizar a `"Error interno"` en producciГіn.

### A07 вҖ” Identification and Authentication Failures В· рҹ”ҙ CRITICAL

**H-07.1 рҹ”ҙ Admin login sin rate limit ni lockout**

Ya descrito en H-04.1. Agrava el problema de H-02.1 вҖ” incluso si el password se hashea, un atacante con diccionario puede intentar 1000 passwords/segundo sin consecuencias.

**Fix:** mismo patrГіn de `@upstash/ratelimit`. 5 intentos por IP + email por 15 min. Respuesta 429 + log del intento.

**H-07.2 рҹҹЎ Cookie `secure` condicionada a NODE_ENV**

Archivo [lib/admin-auth.ts:20](lib/admin-auth.ts):
```ts
secure: process.env.NODE_ENV === 'production',
```

Correcto para desarrollo local (HTTP) pero fiar del env var es frГЎgil. Un deploy de preview sin `NODE_ENV=production` enviarГӯa cookie sin `Secure`. Hardcodear `secure: true` tras deploy inicial.

### A09 вҖ” Security Logging & Monitoring В· рҹҹЎ MODERATE

**H-09.1 рҹҹЎ Sin integraciГіn de error tracking**

No hay Sentry/Logtail/similar. Errores solo en `console.error()` que Vercel captura pero sin agrupamiento ni alertas.

**Fix recomendado:** `npm install @sentry/nextjs`, configurar DSN en Vercel env. Plan free 5K eventos/mes.

**H-09.2 рҹҹЎ Sin audit trail de cambios admin**

`/api/admin/precios` PATCH actualiza tasas sin loggear quiГ©n/cuГЎndo/quГ© cambiГі. Si alguien compromete la cuenta admin y envenena precios, no hay rastro.

**Fix recomendado:** tabla `admin_audit_log` con columnas `admin_email`, `action`, `entity`, `before`, `after`, `created_at`. Insertar en cada PATCH/POST de admin.
